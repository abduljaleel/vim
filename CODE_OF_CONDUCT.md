# Code of Conduct

VIM adopts the [Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) as our Code of Conduct. The full text is authoritative and available at that URL.

## Summary

We are committed to providing a welcoming, respectful, and harassment-free environment for everyone who participates in the VIM community, regardless of background, experience level, or identity.

## Scope

This Code applies to all project spaces, including:
- GitHub repositories, issues, pull requests, and discussions
- OpenSSF Slack channels and mailing lists
- Community calls and meetings
- Conferences, meetups, and events where someone represents the project

It also applies when an individual is officially representing the project in public spaces.

## Reporting

If you experience or witness behavior that violates this Code, please report it by emailing:

**conduct@vim-project.org** (monitored by the Code of Conduct Committee)

Reports will be:
- Reviewed and investigated promptly
- Kept confidential to the extent possible
- Handled with respect for the privacy and security of the reporter

If a report involves a member of the Code of Conduct Committee, that member will recuse themselves from the investigation. If no impartial committee member is available, the report will be escalated to the OpenSSF Technical Advisory Council (post-sandbox acceptance).

## Enforcement

The Code of Conduct Committee will follow the [Contributor Covenant Enforcement Guidelines](https://www.contributor-covenant.org/version/2/1/code_of_conduct/#enforcement-guidelines) when determining consequences. Possible responses include:

1. **Correction** — Private, written warning with clarification of expectations
2. **Warning** — Formal warning with consequences for continued behavior
3. **Temporary Ban** — Temporary exclusion from project spaces
4. **Permanent Ban** — Permanent exclusion from all project spaces
